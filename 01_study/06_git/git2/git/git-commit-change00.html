$ git commit --amend
커밋 기록을 바꾸기 전에 주의해야할 사항이 있다. remote 저장소에 push 안된 상태일 때, 즉 자기 자신의 local 저장소에만 commit이 되어있을 때 이걸 사용하자. 그리고 remote 저장소에 push 되었어도 아무도 git pull을 안받았다는 것이 확실하다면 이걸 사용하자. 정말로 어쩔 수 없이 remote 저장소에 push된 경우임에도 불구하고, commit 기록을 변경해야된다면, 같이 작업하는 작업자분들에게 양해를 구하자. 안 그러면 remote 저장소와 충돌할 확률이 크다. ########################################################################################################## 바로 이전 커밋 기록을 바꾸는 명령어는 단순하다. 만약 커밋 메시지 뿐만아니라 커밋 내용도 추가되어야 하는 거라면, git add (파일이름) 명령어를 먼저 실행하여 직전 커밋에 추가되어야할 내용을 stage에 올린다. 그리고 그 후에 git commit --amend 위 명령어를 입력한 후, 커밋 메시지를 작성하여 커밋하면 직전 커밋 내용에 같이 함께 커밋된다. 커밋 메시지도 함께 바뀐다. 이 과정에서 당연히 마지막 커밋 번호(SHA)는 바뀌게 된다. (git commit --amend 명령어 실행 전과 후가 다르다.) 위에서 말했듯이 remote 저장소에 이미 push된 commit 기록을 변경한 거라면, push할 때, git push (저장소이름) (브랜치이름) -f 위 명령어를 사용하자. -f는 '강제로 한다'는 옵션이다. ##########################################################################################################
