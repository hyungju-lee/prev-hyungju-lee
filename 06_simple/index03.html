HTML, CSS, JavaScript란 무엇인가요?

HTML, CSS, JavaScript
갖다놓고(HTML), 꾸미고(CSS), 시킨다(JavaScript).
오늘날의 HTML, CSS, JavaScript는 웹사이트, 즉 브라우저에서 동작하는 소프트웨어에만 국한되지 않습니다.
React Native나 Native Script 처럼 이것들, 혹은 비슷한 변형들로 모바일 앱을 만드는 기술들이 이미 널리 사용되고 있고
심지어 Electron.js 같은 걸로 웹사이트가 아닌 컴퓨터 프로그램까지 이젠 위 셋(HTML, CSS, Javascript)로 만들 수가 있습니다.
Javascript
원래는 브라우저에서 웹사이트를 돌리는 목적으로 만들어진, 그닥 대우도 잘 못 받는 언어였는데, 계속해서 발전을 하고,
특히 Node.js가 이걸 브라우저 바깥 세상으로 꺼내오면서 위상이 확 올라갔습니다.
웹사이트에서 돌아가는 자바스크립트는 브라우저에서 다양한 일을 수행하고, HTML로 올려놓은 요소들을 변형시키거나 직접 만들어내기까지 합니다.

자바스크립트는 지금도 빠른 속도로 발전해나가는 언어라 끊임없이 공부할게 생길 것입니다.
예전에는 대부분이 jQuery(write less, do more)라는 강력한 라이브러리로
순수 자바스크립트보다 훨씬 코딩을 편하고 쉽게 하곤 했는데
요즘은 자바스크립트도 파워풀해지고, jQuery의 성능 문제도 있고
프론트앤드 프레임워크 같은 새로운 개발방식(angular, vue, react)들이 나오면서 "jQuery는 배우지 않아도 된다."
심지어 "이제는 쓰지 않도록 해야 한다." 는 의견들이 많이 보입니다.
입문자들은 우선 자바스크립트를 탄탄히 배워놓고 이런 라이브러리나 프레임워크를 필요에 따라 배워나가면 더 좋을 겁니다.
